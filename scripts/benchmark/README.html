LeIsaac PickOrange — Benchmark Infrastructure

scripts/benchmark/ — Reusable closed-loop evaluation pipeline for VLA policies on LeIsaac SO-101 PickOrange.

1. 这个目录是干嘛的 / Purpose

给 LeIsaac PickOrange 任务做 统一的、可复现的闭环评测。一行命令跑一个 baseline，自动开/关 server + 采集 GPU + 写 metrics JSON + 聚合成 README leaderboard。

TL;DR: One pipeline to run any policy through ≥5 rounds of closed-loop sim eval, collect GPU stats, and rebuild the leaderboard.

STUCK_WINDOW_S=30 STUCK_EPS_RAD=0.05 \

    n17-hispace \

    "GR00T-N1.7 (hi-space, h=40)"

6. 各模型的 best action_horizon / Per-model best h

VLA 的 chunk-policy 必须用接近训练时的 chunk 长度推理。硬编码 h=16 跑所有模型是历史最大坑（N1.6 训练 h=50 被截到 16，效果减半）。

Modelh_trainh_use验证方式

Default lookup: baselines.tsv column 3 (matched on column 4 ckpt), then fall back to config.json#action_horizon via HF Hub.

7. 早期终止信号 / Early-exit signals

三个并列 trigger 让 episode 在 wall_cap 之前正常结束，节省时间且让 avg_round_s 真实反映 policy operation time：

Signal触发条件Reason
env_success3 颗都在盘 + arm 归位Task done — env-side
stuckround > 30s 后 arm 5 个关节 5s 范围 < 0.05 rad"Arm gave up" — quasi-static
home_returnarm 曾离 home 0.5 rad+，再回到 home 0.15 rad 内"Arm retracted" — close to start pose
wall_capround_age ≥ 180sSafety net

所有 trigger 都用 arm joints 0-4，忽略 gripper（gripper 自循环开合会污染信号）。

8. 严格 placement 计数 / Strict orange-in-plate check

env 内部 success 判定较宽（10cm 方框），我们额外的 strict cylindrical check 在每帧验证：

维度阈值语义
xy_dist (盘心 → 橙心)< 0.10 m对齐 env 10cm 容差
dz (盘心 → 橙心)0.005 – 0.20 m上限 20cm 允许 3 层叠
velocity< 0.05 m/s橙子已 settle，非夹放中
plate up_z> 0.7盘子没翻
snapshot timingenv.step() 之前避开 auto-reset 假阴
历史坑 / Past bugs
  • reset 假阴: 旧代码在 env.step() 之后读 placed_flags，单 env 模式下触发 termination 时 obs 已 auto-reset 成新随机橙位 → 严格 0/9 假阴
  • dz_max=0.05 漏叠: 5cm 上限漏判堆叠橙；放宽到 0.20
  • plate_r=0.08 紧: 比 env 10cm 还紧；放宽到 0.10
  • sticky-OR 假阳: 旧版本累计 OR 把"放完又飞"也算 placed；改成 pre-step snapshot

VLA 提升方法 — 待验证清单

用 X-VLA 做实验台 · 2026-05-19 · 三模型协作（Opus 4.7 / Codex gpt-5.4 / Opencode deepseek-v4-pro）

📌 核心思路 / The pitch
X-VLA 在 4090 上 10k step ≈ 17 min（batch=8, n_obs=2），是当前所有 VLA 中最快的训练-到-eval 循环（OpenVLA-7B ≈ 4h, GR00T ≈ 2h, π0.5 ≈ 1.5h）。
👉 用 X-VLA 当实验试验台，快速筛选哪些方法真有效，再迁移到大模型上少走弯路。

当前 baseline (Path 1, GRIPPER=5 + n_obs=2 + batch=8)：7k ckpt = 1/9 oranges (3 round)。
2026-05-19 update：Path 2 Body-desc prompt 经 Phase A+B 系统对比 已证伪，详见 §2 已证伪。Phase A 看到的 2/6 是 single-round variance。
⚠️ 方法学教训（2026-05-19）：单 round eval 数据不可信。X-VLA + RF action head + 7k 训练 = 高方差推理，初始姿态决定 episode 命运比 model capacity 影响更大。所有后续实验必须 ≥3 完整 round 且不按 R 才算数。

0. TL;DR

本文目的：列出可在 X-VLA 上 ≤2h 验证的所有 VLA 提升方法，每条带预期增益、实现成本、迁移性。Path 2 phase B 训练完成后逐条筛选执行。

排序原则：(预期 closed-loop 增益) × (迁移到其他 VLA 的潜力) / (实现成本) 降序

方法学根基：
  • val_loss 不预测 closed-loop（已多次证伪，见 compact-resume-2026-05-19 记忆）→ 所有方法只能用 closed-loop sweep 验证
  • X-VLA 学习曲线 narrow sweet spot (~1k step 宽)→ 每种方法必须做 sweep，不能单点评估
  • 3 个模型独立调研后交叉验证 → 减少单模型 hallucination 风险

⚠️ Prompt 偏好与训练 loss 紧耦合 (2026-05-20):
"Pick up the orange and put it in the plate" 在 baseline 上似乎略优 (37% vs default 33%, 噪声内)，但在 L1 model 上 反而 hurt:
  • L1 10k + default prompt = 7/18 (39%)
  • L1 10k + "Pick up...put...in..." = 2/18 (11%) — 掉 5 oranges!
"Best prompt" 是模型相关的，不能跨 retrain 配置盲转。EN: Prompt preference is loss-conditioned — what helps baseline harms L1 retrain.
真实方差分析 (2026-05-20):
  • 3-round closed-loop eval 方差极大: 同 cfg 跨 sweep 可从 1/9 跳到 5/9
  • X-VLA + RF action head + sim 物理初始随机 = 单 episode 命运由初始姿态主导 > model capacity
  • 必须 6 round (18 ep) 以上才能区分 ±10% per-ep 差异
  • 本次 session 修正了 3-round 误信号（如 EMA α=0.3 = 5/9 神话）

结论：6 项实验全部 6-round 校验后：

  • 真改进只有 1 项：n_action_steps 8→32 → 7/18 (39%)。整 chunk 复用比每步重规划好。
  • 5 项有害或无效：EMA / TAE / velocity-reweight / image-aug / body-desc 全部下降。
  • baseline 7k @ h=32 是 X-VLA + 50-demo + Path 1 配置下的 inference-time 最优。
  • 下一步要突破 39% 上限需要：(1) 扩 dataset 突破 50-demo 天花板; (2) 切换更适合 sim 的 backbone (e.g. ACT/π0.5 都跑过)；(3) Synthetic DAgger 修 covariate shift。

  • ❌ Online RL fine-tune（Codex 24-72h; Opencode VLA-RL score 0.1, RL-100 score 1.7）
  • ❌ Pretrain from scratch on OXE/DROID（Codex Medium; Opencode 40-80h, score 0.8）
仅 1 家强烈推荐（中置信度，待 sweep 验证）：
  • 🟡 Mixture of Horizons (Codex 2511.19433): π0.5 + MoH = 99% LIBERO，与 X-VLA RF head 最契合；Opencode 没列。值得 8-16h 试。
  • 🟡 Diffusion-DAgger (Codex 2402.17768): 8 demos 20→80%，但需要 Isaac Sim 接 teleop。Opencode 列 Diff-DAgger score 2.1（成本太高）。
  • 🟡 MAP-VLA 检索 (Codex 2511.09516): X-VLA soft prompt 顺手；Opencode 列 Retrieval-aug score 1.0（迁移性 MED）。
  • 🟡 LoRA fine-tuning (Opencode #7 score 18.8): Codex 没单独列。X-VLA 当前 full FT，加 LoRA 可保留预训练权重防过拟合。

4. 验证流程（每个方法）

关键约束：

  • 每方法只动 1 变量（vs Path 1 同时改 3 个的反面教训）
  • 必须 sweep 全 ckpt 范围，不能单点（narrow sweet spot 现象）
  • 每方法跑 3 round 完整 60s，不能 ep1 提前 R 跳
  • baseline 与新方法用同种子（dataset.episodes 一致, val split 一致）

5. 实验队列 — Opus+Codex 合并优先级

排序原则：低成本 P1 先做（先剪枝再投入）。Codex 的发现：velocity-reweight 和 TAE 是最便宜的两条，应紧接 Phase B sweep。

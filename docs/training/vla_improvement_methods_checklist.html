VLA 提升方法 — 待验证清单

用 X-VLA 做实验台 · 2026-05-19 · 三模型协作（Opus 4.7 / Codex gpt-5.4 / Opencode deepseek-v4-pro）

📌 核心思路 / The pitch
X-VLA 在 4090 上 10k step ≈ 17 min（batch=8, n_obs=2），是当前所有 VLA 中最快的训练-到-eval 循环（OpenVLA-7B ≈ 4h, GR00T ≈ 2h, π0.5 ≈ 1.5h）。
👉 用 X-VLA 当实验试验台，快速筛选哪些方法真有效，再迁移到大模型上少走弯路。

当前 baseline (Path 1, GRIPPER=5 + n_obs=2 + batch=8)：7k ckpt = 1/9 oranges (3 round)。
2026-05-19 update：Path 2 Body-desc prompt 经 Phase A+B 系统对比 已证伪，详见 §2 已证伪。Phase A 看到的 2/6 是 single-round variance。
⚠️ 方法学教训（2026-05-19）：单 round eval 数据不可信。X-VLA + RF action head + 7k 训练 = 高方差推理，初始姿态决定 episode 命运比 model capacity 影响更大。所有后续实验必须 ≥3 完整 round 且不按 R 才算数。

0. TL;DR

本文目的：列出可在 X-VLA 上 ≤2h 验证的所有 VLA 提升方法，每条带预期增益、实现成本、迁移性。Path 2 phase B 训练完成后逐条筛选执行。

排序原则：(预期 closed-loop 增益) × (迁移到其他 VLA 的潜力) / (实现成本) 降序

方法学根基：
  • val_loss 不预测 closed-loop（已多次证伪，见 compact-resume-2026-05-19 记忆）→ 所有方法只能用 closed-loop sweep 验证
  • X-VLA 学习曲线 narrow sweet spot (~1k step 宽)→ 每种方法必须做 sweep，不能单点评估
  • 3 个模型独立调研后交叉验证 → 减少单模型 hallucination 风险

2. 已完成 / 已证伪

实验结果结论
GRIPPER_SCALE 3→5 + n_obs 1→2 + batch 4→8 (Path 1) sweet spot 10k→7k 左移，天花板仍 1/3 step 数提早，单 round 上限未破
Offline action-MSE eval v3 (teacher-forced + gripper timing) v3 best ckpt = GUI 0/3，GUI best ckpt = v3 中位 val_loss / offline MSE 不预测 closed-loop（多次证伪）
Auto closed-loop sweep (Path 4) 10 ckpt × 3 round × 60s 唯一可靠 ckpt 选择手段，但每 ckpt ~5 min wall
Body-desc inference-time prompt (Phase A, 7k ckpt) 2/6 oranges, 单 round 2 颗 看似突破，但 Phase B retrain 不能复现 → 是 variance
❌ Body-desc 进 dataset task field retrain (Path 2 Phase B, 12k ckpts × 短/长 prompt inference) 各组合均无突破 baseline 1/9 已证伪：Florence2 language encoder freeze 下，长 prompt 只是 token embedding 微扰，不改变 action conditioning 分布。详见 memory/xvla-bodydesc-negative.md

3.4 三模型交叉验证（共识）

三家一致同意（高置信度，应立即做）：
  1. Action chunking horizon tuning（Opus P1 + Codex 提及基本盘 + Opencode #1, score 46.9）
  2. Temporal Ensembling / Action Smoothing（Opus P1 + Codex 基本盘 + Opencode #2 + #6）
  3. Body description prompt（Opus P1 + Codex Embodiment Transfer + Opencode #3, score 45）→ 本项目正在做
  4. Per-channel loss weighting（Opus P2 已做 + Codex AttenA+ + Opencode #4, score 22.5）→ 本项目 GRIPPER=5 已做
  5. Visuomotor / lighting aug（Opus P1 + Codex 38.75% + Opencode #5）
三家一致避免：
  • ❌ Explicit CoT for action（Codex 2602.01067 报告 no improvement; Opencode score 2.5）
  • ❌ FAST tokenizer 替换 RF head（Codex Low-Med 迁移；Opencode score 2.9，侵入性大）
  • ❌ Online RL fine-tune（Codex 24-72h; Opencode VLA-RL score 0.1, RL-100 score 1.7）
  • ❌ Pretrain from scratch on OXE/DROID（Codex Medium; Opencode 40-80h, score 0.8）
仅 1 家强烈推荐（中置信度，待 sweep 验证）：
  • 🟡 Mixture of Horizons (Codex 2511.19433): π0.5 + MoH = 99% LIBERO，与 X-VLA RF head 最契合；Opencode 没列。值得 8-16h 试。
  • 🟡 Diffusion-DAgger (Codex 2402.17768): 8 demos 20→80%，但需要 Isaac Sim 接 teleop。Opencode 列 Diff-DAgger score 2.1（成本太高）。
  • 🟡 MAP-VLA 检索 (Codex 2511.09516): X-VLA soft prompt 顺手；Opencode 列 Retrieval-aug score 1.0（迁移性 MED）。
  • 🟡 LoRA fine-tuning (Opencode #7 score 18.8): Codex 没单独列。X-VLA 当前 full FT，加 LoRA 可保留预训练权重防过拟合。

4. 验证流程（每个方法）

关键约束：

  • 每方法只动 1 变量（vs Path 1 同时改 3 个的反面教训）
  • 必须 sweep 全 ckpt 范围，不能单点（narrow sweet spot 现象）
  • 每方法跑 3 round 完整 60s，不能 ep1 提前 R 跳
  • baseline 与新方法用同种子（dataset.episodes 一致, val split 一致）

5. 实验队列 — Opus+Codex 合并优先级

排序原则：低成本 P1 先做（先剪枝再投入）。Codex 的发现：velocity-reweight 和 TAE 是最便宜的两条，应紧接 Phase B sweep。

#实验预期增益时间预算依赖
1 Phase B sweep（body-desc retrain ckpts 4k-8k） 验证 Phase A 2/6 信号是否 retrain 后稳定 ~25 min 无 ✅ 训练完毕
2 Action chunking horizon sweep（n_action_steps ∈ {4, 8, 16, 32}，在最好 ckpt 上 inference） ±15% by horizon 1h（无 retrain） 取 Phase B 最佳 ckpt
3 Velocity-aware loss reweighting (AttenA+, 2605.13548) +1-2% but 极便宜 2-4h 改 action_spaces.py + 17 min retrain + 25 min sweep 从 dataset 算 velocity 阈值
4 TAE (Temporal Action Ensembling, 2304.13705) +10-15% smoothness 3h server 端 exp-blend buffer + 25 min sweep 无 retrain
5 Mixture of Horizons (MoH, 2511.19433) π0.5 + MoH = 99% LIBERO；对 RF head 最契合 8-16h 实现 dynamic chunk length
6 Lighting/image aug (2508.01442) +38% unseen lighting 8h aug pipeline + retrain + sweep ≈ 10h 无
7 TAS (Temporal Action Selection, 2511.04421) +73 abs success（如果真） 10-20h 需多时刻 chunk cache + selector net
8 Synthetic DAgger (2402.17768) Pushing 20→80% with 8 demos 8-16h Isaac Sim teleop + relabel pipeline
9 MAP-VLA retrieval (2511.09516) Real +25, sim +7 8-16h X-VLA soft prompt 改 memory bank
10 DROID co-training (2403.12945) +14.8 abs sim 16-32h DROID dataset 下载 + 筛选相近 task
⚠️ 不要做（已证伪 / 性价比低）：
  • ❌ Explicit CoT for action（2602.01067 报告 no improvement）
  • ❌ FAST tokenizer 替换 RF head（侵入性大，不是当前 X-VLA 路径）
  • ❌ Online RL fine-tune（工程量 24-72h, +4.5 abs 性价比差）
  • ❌ Offline action-MSE eval（本项目多次证伪）

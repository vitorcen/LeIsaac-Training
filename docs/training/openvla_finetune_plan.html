4. 推理实时性：实测数据 · Inference Latency (measured)

已实测 · Measured on RTX 4090, bnb NF4 4-bit, no flash-attn：
负载LatencyGPU
加载模型 (cold)9-32 s4.38 GB
第 1 次 inference (cold, KV miss)~440 ms4.38 GB
稳态 inference (warm)233-242 ms / action4.38 GB
原 480ms 估算过悲观（基于无 KV-cache 假设）。实测 240ms 是含 vision prefill 的 6-token AR decode 完整耗时。

4.1 30Hz 闭环可行性 · 30Hz feasibility

240 ms / action @ 30Hz (33ms/step) = slowdown ~7×。Isaac Sim 不能跟 30Hz 实时跑，但：

#方案实际效果成本
1step_hz 降到 5 Hz每步 200 ms 推理窗口 ≈ 1× 实时；dataset 30Hz → distribution mismatch 待 fine-tune 救低
2切到 OpenVLA-OFT (chunk + parallel decode)论文报告 25× 加速；本质换 model 重训高
3TensorRT-LLM 部署~10ms/token = 60ms/action ≈ 2× slowdown中
常见误区 · Common myth: bnb int4 不会加速 decode（实测中 4-bit 与 bf16 速度持平）；它只省显存。真正的加速需要 chunked / parallel decode 架构（OpenVLA-OFT, PD-VLA）或 TensorRT-LLM 引擎。
bnb int4 saves memory, NOT compute. For real speedup, use OFT-style chunked decode or TensorRT-LLM.

6. 实施路径 · Implementation Phases

Phase 1 — Inference server scaffold (~2h)

复用 server/serve_pi05.sh + pi05_leisaac/server.py 模板：

  • server/serve_openvla.sh — 启动脚本
  • server/openvla_leisaac/server.py — ZMQ + msgpack, wire-compatible with Pi05ServicePolicyClient
  • server/openvla_leisaac/__init__.py, pyproject.toml

关键改动 vs π0.5 server：

  • 加载 openvla/openvla-7b（HF AutoModelForVision2Seq，trust_remote_code=True）
  • 每次 get_action 请求只返回 1 个 action（shape (1, 6)）
  • config: load_in_4bit=True 默认开

Phase 2 — Client 端补丁 (~1h)

修改 LeIsaac/source/leisaac/leisaac/policy/service_policy_clients.py：

  • 新增 OpenVLAServicePolicyClient（参照 Pi05ServicePolicyClient）
  • n_action_steps=1（不是 chunk）
  • policy_inference.py 加 --policy_type=openvla 分支
  • horizon 在 OpenVLA 永远 = 1，可忽略 / 抹平

Phase 3 — 数据转换 (~1h)

OpenVLA 官方训练用 RLDS / TFDS。我们的 LeIsaac dataset 是 LeRobot v3.0 (parquet)。两种选择：

  • A: 用 LeRobot 包 OpenVLA（优先） — 检查 LeRobot 是否已有 openvla policy 集成；若有，复用现有 lerobot_finetune.sh
  • B: convert LeRobot v3.0 → RLDS（耗时但可用 openvla 原生训练）

Phase 4 — LoRA 微调 (~12h，含反复跑)

新建 scripts/training/openvla/train.sh（类比 pi05 的 train.sh）：

--lora_rank=32 --lora_alpha=16 --lora_target_modules q_proj,v_proj
--batch_size=4 --lr=5e-5 --grad_accum=2 --gradient_checkpointing=true
--steps=10000 --save_freq=1000
--bf16 --image_aug=true

Phase 5 — Eval + benchmark (~2h)

接到 LeIsaac eval pipeline：

  • step_hz 必须降到 5-10（实时性约束）
  • 观察：实际推理延迟、success rate（单 round + 10-round 平均）
  • 跟其他 4 个 baseline 同条件对比

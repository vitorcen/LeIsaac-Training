StarVLA GR00T_v2 动作头：把 N1.5 血统的头升级到 N1.7 设计

Upgrading starVLA's N1.5-lineage QwenGR00T action head to the GR00T N1.7 head design, then training StarVLA-Qwen3-VL-8B-GR00T_v2-PickOrange to challenge the PI_v3 entry (63.3%) and close in on the N1.7 SOTA (68.3%).

2026-06-12 · LeIsaac/docs/training · 代码落点：vitorcen/StarVLA fork（starVLA_dev 分支，只加新文件不改上游）· 评审：codex (gpt-5.5) + mimo-v2.5-pro，见 §10

TL;DR　starVLA 的 QwenGR00T 头是 GR00T N1.5 的瘦身移植（DiT-B 768×16L、全 VL token 盲 cross-attn、32 个 future tokens、无 VLLN、无 state dropout），同骨干下被自家 PI_v3 头压 10 个点（53.3% vs 63.3%）。 N1.7 SOTA ckpt（68.3%）的头实测配置是：DiT 16L × inner 1536 + AlternateVLDiT（图/文 token 交替 cross-attn）+ VLLN + state_dropout 0.2 + 去掉 future tokens。 GR00T_v2 = 把这五件事搬进 starVLA（新文件 QwenGR00T_N17.py + GR00T_N17_ActionHeader.py，全部 config 开关化），冻结 Qwen3-VL-8B 重训。 两段式跑法（双评审一致建议，§10）：先跑 v2-lite（VLLN+删 future tokens+state_dropout，DiT-B 不扩，~30k 步便宜验证统计/正则收益），再跑 v2-full（+AlternateVLDiT+1536，头实测 612M，与 PI_v3 的 634M 几乎同参数——公平对比成立）——同一份头实现，两份 yaml。
EN: starVLA's QwenGR00T head is a slimmed N1.5 port. The actual N1.7 SOTA checkpoint uses a 16-layer/1536-dim DiT with image/text-alternating cross-attention, VL LayerNorm, state dropout 0.2, and no future tokens. GR00T_v2 ports these deltas as NEW config-gated files in the fork. Per both external reviews, roll out in two stages: v2-lite (cheap 3-delta run, DiT-B unchanged) then v2-full (~0.7B head, sized to match PI_v3).

核心洞察 / Key insight：N1.6→N1.7 NVIDIA 自己把 DiT 从 32 层砍回 16 层、删掉 future tokens、删掉 4 层 post-VLM adapter——但加了 AlternateVLDiT + state dropout，结果 PickOrange 上 N1.7(68.3%) > N1.6。方向不是堆深度，是改 VL 条件质量 + 正则。这正合本仓品味：v2 不是"更大的头"，是"消除两个坏特殊情况"（裸 hidden 无规范化、cross-attn 不分图文盲扫）。 NVIDIA themselves went 32→16 layers from N1.6 to N1.7 while adding alternating VL attention and state dropout — the win came from conditioning quality and regularization, not depth.

4. 设计决策清单：N1.7 的每一项，采纳还是不采纳 / Adopt-or-skip ledger

N1.7 机制决策理由（一句话说清）
AlternateVLDiT（图/文交替 cross-attn）采纳v1 最大的坏味道：448² 双相机 ≈ 数百图 token 淹没十几个文 token，盲扫 cross-attn 等于指令信号被稀释；交替路由让两路各有专属带宽。image_mask 可直接从 input_ids == image_token_id 得到，零额外成本。
VLLN（VL 特征 LayerNorm）采纳骨干冻结 → 头无法适配 VLM 输出的尺度漂移，一个 LayerNorm(4096) 即消除该特殊情况；8K 参数，免费。
DiT inner 768→1536（32头×48），16 层采纳对齐 N1.7-PickOrange SOTA ckpt 实测值；头 ~0.7B 与 PI_v3 0.63B 同量级 → 赢了不能被"参数更多"解释。
删 future tokens（32 个）采纳N1.6/N1.7 都已删；它们是任务无关的可学习前缀，60 demo 小数据上是纯过拟合面。
state_dropout 0.2采纳（只做特征层一遍）SO-101 6 维 proprio 在小数据上易成捷径特征；20% 置零逼头依赖视觉。取 N1.7 微调值 0.2，不取预训练值 0.8。注意 N1.7 实际有两层 dropout（processor 数据侧 + head 特征侧，叠加有效率 36%）——我们只在 state_encoder 输出后按样本整 token 置零、training only，与 gr00t_n1d7.py:206-212 对齐（双评审一致）。
VL self-attention 预处理（4 层）config 开关，默认关N1.7-PickOrange SOTA ckpt 本身没用（Identity）——单任务下 VL 特征大概率不需要额外 self-attn 整理（SONIC-G1 多动作语料才用 4 层）；留 vl_self_attention_cfg 字段供消融，默认 null = 无行为。
CategorySpecific 多 embodiment 权重组不采纳我们只有 SO-101 一个 embodiment，32 套权重选 1 套 = 为不存在的问题加 32 倍参数面；普通 MLP 即可。好品味 = 消除特殊情况，不是搬运特殊情况。
RTC 实时纠正 inpainting不采纳benchmark 是开环 chunk 重放（per-model action_horizon=16），无 RTC 介入点。
state_history_length > 1不采纳N1.7 默认也是 1；与 v1 对照要单变量。
relative action（use_relative_action=True）本期不采纳这是数据侧改动（N1.7 SOTA 的潜在混淆因子之一，见 §8），动了它 head-swap 归因就不干净；列为 v2 之后的独立消融。
4096→低维投影后再 cross-attnconfig 开关，默认直连N1.7 是 backbone 维度直连（2048），我们默认 4096 直连少一个部件；但 4096 与 N1.7 的 2048 尺度不等价（codex 评审点），留 vl_proj_dim 开关（LN+Linear 4096→2048）作为 OOM / 训练不稳时的 P0 fallback。

6. 训练与评测计划 / Training & eval plan

6.5 实测结果 / Actual results（2026-06-13）—— 🟠 负面但混淆未控（结论下调为"待复测"）/ NEGATIVE but CONFOUNDED

初判结论：N1.7 头移植到冻 Qwen3-VL-8B head-only PickOrange 上没带来提升，远不如 N1.5 血统的 v1（13.3% ≪ 53.3%）。
⚠️ 事后三方评审（Fable / codex-gpt5.5 / mimo，2026-06-13）推翻了"这是 N1.7 头设计不迁移的干净负面"这一归因——发现一处移植缺口使本次对比不成立（详见下方评审裁决）。结论从"NEGATIVE（头设计不行）"下调为"INCONCLUSIVE / 待一次单变量复测"。 EN: A 3-model review overturned the clean "N1.7-head-doesn't-transfer" reading: a porting gap (wrong VLM feature layer) confounds the v1-vs-v2 comparison. Downgraded from NEGATIVE to INCONCLUSIVE pending one single-variable re-measure.

训练 + sweep（10k→42.5k 全扫，覆盖 4.7 epoch，本地 GUI 5-round headless 180 quick-screen）：v2-full（QwenGR00T_N17，612M 头，rep=8/bs=4/50k 步，weste 4090，~0.62s/step）训练健康。 sweep 曲线确认 N17 是早峰型——峰在 step-17500（1.9 epoch）quick-screen 53.3%，22500=33.3% 次高，之后 25k–42.5k 一路低位（6.7–26.7%，40000 甚至卡死 0%），无二次爬升晚峰（37500 的 33.3% 是噪声）。早峰印证了 §2 的预判（N17 大头比 v1 的 3.3-epoch 峰更早），但峰质量差。

为什么没赢 / Why it failed：§7 把 VLLN 排为预期收益最大，但实测整套 N1.7 deltas（VLLN + AlternateVLDiT + state_dropout + 无 future tokens）在冻 VLM + 60-demo 这套约束下学不出来。NVIDIA N1.7 在机器人上赢，靠的是 Cosmos-Reason2 具身预训练骨干 + 头的机器人预训练 + relative action（§8 列的混淆因子，都是数据/骨干侧），纯头设计移植无法复刻。同骨干下真正赢 v1 的是 PI_v3 的 LayerwiseFM（per-layer cross-attn + 状态离散进文本），不是 N1.7 的这几个 delta。 EN: The N1.7 advantage is data/backbone-side (Cosmos embodied pretraining + robot-pretrained head + relative action), not the head-design deltas — which don't transfer to a frozen-VLM head-only setup. 📋 三方独立评审裁决 / 3-model review verdict（Fable + codex-gpt5.5 + mimo，2026-06-13）
评审目标：13.3% ≪ 53.3% 究竟是真发现还是实现 bug / 移植缺口。逐行核对了头、框架接缝、与真 Isaac-GR00T N1.7 参考实现。结论收敛于三点共识：
  • ① [移植缺口·最可能根因] VLM 特征取层错位：hidden_states[-1]（第 36 层）vs 真 N1.7 select_layer=12（中层）。Fable + mimo 一致首要。真 N1.7（qwen3_backbone.py）物理 pop 掉第 13–36 层、头吃第 12 层中层特征；fork 取最后一层。最后一层特征为 next-token 语言预测优化，且 image 位置的 hidden 到末层已丢失视觉语义——而 AlternateVLDiT 有 4 个 cross block 只准看 image 位置 token（强耦合），等于被 mask 强制喂垃圾。这是静默退化、不崩，对 v2 的杀伤远大于 v1（v1 全 token cross-attn 可绕开坏位）。本次"N1.7 头"从未在正确特征层上测过。
  • ② [真 bug] 部分解冻当前是静默 no-op（Fable + codex + 代码实锤；mimo 误判为无 bug，其自身 trace 自相矛盾）。build_param_lr_groups 按 freeze_modules='qwen_vl_interface' 把整个 VLM 的 param 全列入 frozen 集合并从所有 optimizer group 剔除（trainer_tools.py:132/140）；optimizer 在 main() 中先于 prepare_training() 的 freeze+unfreeze 构建（train_starvla.py:462 vs :473）。之后 apply_partial_unfreeze 翻 requires_grad=True 的那些 param 已不在任何 optimizer group → optimizer.step() 永不更新它们。照现状跑 unfreeze4 = 又一次 head-only，白烧整轮。（§11 line 340 旧注"时序正确"是错的，已更正。）
  • ③ [次生 bug，修②后才暴露] 解冻层裸 bf16 训练、无 fp32 master copy。真 N1.7 把 trainable backbone 参数 cast 回 fp32（backbone_trainable_params_fp32）；lr=1e-5 对 bf16 权重的相对更新贴着 ULP，大量更新静默量化为零。（DeepSpeed ZeRO-2 会兜底 fp32 master，但当前非 DeepSpeed 路径有风险。）
无 bug 项（评审一致）：AlternateVLDiT 路由 / image_mask 构建 / repeat-lockstep / VLLN+dim 对齐 / flow-matching 数学 / state-dropout / 去 future tokens —— 均忠实移植，无实现 bug。Brief 担心的"Qwen3-VL token 布局 ≠ Eagle/Cosmos 致路由失效"不成立（Cosmos-Reason2 本身就是 Qwen3-VL 架构）。

🎯 裁决：① 把"N1.7 头设计不迁移"这一结论撤回，降为"不完整移植（错特征层 + 5 个 delta 同开 + 推理延迟压低绝对值）下显著差于 v1"。② 部分解冻不是当前该跑的实验——v1 用一模一样的 100% 冻结通用 Qwen3-VL 拿了 53.3%，"冻结通用 VLM"解释不了 v1→v2 的暴跌（这才是要解释的现象）。最高 ROI 的单变量决定性实验 = 把 hidden_states[-1] 改成中层（≈第 12 层）后重跑 head-only：若从 13% 回到 ≥ v1，则负面是移植窟窿、非头设计；若仍低，再跑已写好的 v2-lite 做 delta 归因。解冻须先修 ② + ③ 才值得排队，且应同时给 v1 头跑解冻对照，否则仍无法归因。 EN verdict: Retract "N1.7 head doesn't transfer." The decisive, cheap next experiment is mid-layer feature select ([-1]→layer-12) head-only, NOT unfreeze (which is a silent no-op as wired + bf16-lossy, and can't explain a v1→v2 drop both share the same frozen generic VLM).

7. 为什么 v2 应该赢 v1 —— 机制论证与收益排序 / Why it should win（事后看：预期落空）

按双评审收敛后的预期收益排序（两位评审独立给出几乎相同的顺序）：

  1. VLLN（预期收益最大）：冻结 VLM 的 hidden 不为从零训的头做任何尺度保证，4096 维裸喂是最根本的分布错配；一个 LayerNorm 8K 参数直接修输入统计。PI_v3 的 per-layer projector 内含 LayerNorm，干的是同一件事——这可能是它赢 v1 的一半原因。
  2. 删 future tokens：32 个任务无关可学习前缀在 60-demo 上是纯过拟合面，N1.6/N1.7 都已删；免费，但别指望它单独值 10 个点。
  3. AlternateVLDiT（收益中等，初稿高估处已修正）：448² 双相机出 ~600+ 图 token、文 token 仅 ~20。但 PickOrange 是单任务，指令是常量——此项的真实收益是"图 token 条件不被 padding/文本稀释"的质量提升，而非语言理解跃迁（双评审一致指出初稿的"指令带宽"论证在单任务下高估，预计 1–3 点）。
  4. state_dropout 0.2（辅助正则）：6 维 proprio 的捷径风险存在但 PickOrange 重心在视觉定位，预计 0–2 点。
  5. DiT 768→1536（收益最不确定、风险最大）：砍半 5.3× 压缩瓶颈、0.7B ≈ PI_v3 0.63B 排除参数量解释——但 3× 参数在 60-demo 上同样可能把 v1 的 30k 悬崖提前。这正是 v2-lite 先行的原因：先确认 1+2+4 的统计/正则收益，扩容的归因才干净。

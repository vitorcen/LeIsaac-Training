3. 候选模型分级与推荐 · Candidate prioritization

按"上手成本 × 横评价值"分 3 个梯队：

Tier S — 立即可做，价值最高

#模型为什么值得 · Why预计耗时风险
1openvla-7b（外部） 7B 离散 token VLA 档位空白；单臂 DROID 直接可用；MIT license；详见 openvla_finetune_plan.html 2-3 天（含 RLDS 转换 + 训练 + eval） 实时性 480ms/action — 需 int4 量化或 step_hz=10
2vqbet（LeRobot 原生） 离散动作表示这一档完全空白；ResNet + GPT-2 backbone 小（<200M）；4090 训练快；与 ACT/DP 完美互补 4-6 小时 低
3pi0（LeRobot 原生） 跟我们 π0.5 同家，但更"古典" — 对照 π0 vs π0.5 看 v1→v2 升级是否帮助 LeIsaac；同 LoRA 配方可复用 8-12 小时 低（与 π0.5 流程对齐）

Tier A — 中等改造，对横评有价值

#模型为什么值得 · Why预计耗时风险
4pi0_fast FAST 离散 token 表示（vs π0.5 连续 flow）— 测试 "同 backbone 不同 action head" 影响；与 OpenVLA 都是离散 token VLA 同档对比 10-15 小时 中（FAST tokenizer 配置坑）
5wall_x Qwen2.5-VL backbone（同 LingBot-VLA），新一代 VL；4090 应能塞下；走通可作为"LingBot 的开源替代" 1-2 天 中（新 policy，没社区参考 recipe）
6multi_task_dit DiT 架构（vs UNet1D），填上 "Transformer-based diffusion" 这一档；用同样 dataset 跟 DP 直接对比 6-8 小时 低
7Fast-WAM（外部） 新增 ⭐⭐：填 "Video World-Action Model" 横评空白；6B Wan2.2-DiT + 1B action expert；190ms latency on RTX 5090D 实测，4090 估 ~220ms；ckpt 公开；详见 fastwam_finetune_plan.html 3-5 天（含 QLoRA fine-tune） 中（reverse-engineer 单帧 API + bnb 4-bit 集成）

Tier B — 探索性，需深度改造

#模型为什么值得 · Why预计耗时风险
7xvla Florence2 backbone（Microsoft，多任务 vision-language），独特 VL 路线；扩展横评 "different VL pretrain" 维度 1-2 天 中
8lingbot-vla（外部） 2026-01 蚂蚁开源，Qwen2.5-VL-3B + flow；但仅训过双臂，单臂需 swap action head；详见 lingbot_vla_finetune_plan.html 2-4 天 高（单臂 head 重训 / 等官方）
9groot 自训 之前用的是 LightwheelAI 社区 ckpt；自己用 NVIDIA Isaac-GR00T 训练框架走一遍闭环，得到 fully-controlled baseline 2-3 天 中（GR00T 训练流程复杂）

Tier Skip — 不做

  • sac / tdmpc：纯 RL，需要 reward function + env 闭环，与我们 demo-only dataset 不匹配
  • sarm：是 reward model 不是 policy；如果未来要做 RL 微调 BC 模型可作辅助
  • rtc：是推理 wrapper（实时 chunking），所有 chunk-based policy 都可套用 — 单独不构成 baseline，但可作为 "ACT/DP/π0.5 加速包装" 单独评测一次
  • pi_gemma.py：π0 系列内部组件，非独立 policy
  • DreamZero (NVIDIA)：14B Wan2.1 backbone，4090 推理估 25-30 s/chunk 不可用；QLoRA 训练 4-13 天非停机；Vizuara SO-101 LoRA 的所谓 patches repo 是 paper 网站无 code
2026-05-17 更新：Fast-WAM (6B Wan2.2-DiT) 跑通 DreamZero 想做但 4090 做不到的 "video-world-model 横评条目"。 本质差异：DreamZero imagine-then-execute（推理时多步去噪生成视频）= 慢；Fast-WAM 训练时 co-train video，推理时 SKIP imagination = 快。

⚠️ 2026-06-03 调研修正 · Corrections：
  • multi_task_dit 定位下调：原 2.1 表标"⭐ 低风险 DiT"误导。实测源码 = CLIP ViT-B/16 @224 + 只取 CLS token（整图压成单个 768 维全局向量，无 spatial map），视觉比 DP 的 ResNet 更弱。DiT head ≠ GR00T 的成功来源（GR00T 靠 Eagle@448 强视觉）。60 demo 还落在其文档点名的 collapse 高风险区。
  • OpenVLA-OFT（arXiv 2502.19645）：修好了原版 action head 死因（离散 token→continuous+L1+并行解码，480ms→73ms 解 timeout），但 vision 仍 224 未动 → 逃得出 0% 但天花板被 224 压死，打不过 GR00T。仅"隔离验证 action head 增量"有科研价值。
  • Wall-X 升级为首选新候选：Qwen2.5-VL 动态分辨率根治 224 死穴 + 官方自带 SO-101 单臂示例 + LeRobot 原生 + zero-shot Fruit Sorting 96。详见 wallx_finetune_plan.html。
  • Qwen3-VL 代已不空白：StarVLA（MIT 真 ckpt）/ ABot-M0 / Qwen-VLA。最可操作的是 StarVLA+Qwen3-VL-4B，详见 starvla_qwen3vl_finetune_plan.html。

4. 执行顺序建议 · Recommended order

关键节点 · Key milestones：

  1. 完成 Tier S 后（当前 +3 模型）：横评矩阵达 7 个 policy，覆盖 chunk continuous / discrete token / RL-style coding 三大动作表示
  2. 完成 Tier A 后（再 +3）：横评矩阵 10 模型，同 backbone × 不同 action head（π0 vs π0.5 vs π0-FAST） 三角对照成立
  3. Tier B 看情况：xvla / LingBot 是"探索向"，决定取决于 Tier S/A 跑完后我们想验证的具体 hypothesis

洞察 · Insight（codex review 修正）：VQ-BeT 跟 OpenVLA 不是同一档：

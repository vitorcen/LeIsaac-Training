Fast-WAM 24GB 显存账：bf16 LoRA vs NF4 QLoRA

RTX 4090 24GB 单卡 · LeIsaac SO-101 PickOrange · 2026-05-17

VRAM accounting for fine-tuning Fast-WAM (Wan2.2-5B Video DiT + 1B Action Expert) on a single RTX 4090. Compares plain bf16 LoRA against NF4 QLoRA.

TL;DR
bf16 LoRA 理论 19-21 GB，4090 24GB 紧但能压进去（前提：UMT5 CPU offload + batch=1 + grad_ckpt + 别开 Isaac Sim 共卡）； 任何一项松了就 OOM。NF4 QLoRA 10-11 GB，舒服很多，能容忍 Isaac Sim 共卡 demo。
结论：能纯 LoRA，但 QLoRA 是工程上更稳的默认。
bf16 LoRA technically fits 24GB but only with UMT5-CPU-offload + batch=1 + grad_ckpt and no Isaac Sim co-load — any slack causes OOM. NF4 QLoRA leaves ~13GB headroom. Pure LoRA is feasible; QLoRA is the safer default.

15.7M trainable param 在 bf16 占 ~32 MB；AdamW state (fp32 m+v) 占 8 byte/param ≈ 126 MB；gradients ≈ 32 MB。总训练态 buffer ~190 MB，几乎可以忽略。 Trainable optimizer/grad buffer is <200 MB regardless of quant level — the dominant cost is base weights + activations.

7. 决策建议 · Recommendation

分阶段走 · Phased path：
  1. Phase A — bf16 LoRA 跑通 smoke train（0.5 天）：先证训练 pipeline 通，UMT5 必须 CPU offload，batch=1，跑 100 step loss 看趋势。
    Verify training pipeline first with bf16 LoRA (cheap to set up), UMT5 CPU offload + batch=1.
  2. Phase B — 升级到 NF4 QLoRA 跑正式 10k step（1-2 天）：拿到稳态后切 NF4，腾出 13GB 给 batch=2 加速、给同卡 Isaac Sim eval。
    Upgrade to NF4 QLoRA for the real 10k-step run — free 13GB for batch=2 speedup and same-GPU Isaac Sim eval.

红线 / Hard constraints

  • UMT5 必须 CPU offload。否则 +5GB 直接 OOM (无论 LoRA 还是 QLoRA + Sim 共卡)。FastWAM 训练时 prompt 不变，预计算一次 context_emb 缓存即可。
    UMT5 must be CPU-offloaded (or precomputed once into a context cache). +5GB overhead otherwise OOMs.
  • flash-attn 必须开（2.8.3 已装）。否则 attention score 物化 +1.2GB/layer × 30 layer = 直接爆炸。
  • batch=1 + grad_accum=4。如果想 batch=2，必须 NF4。
  • 同卡 Sim eval（demo 用）：bf16 不行，必须 NF4。

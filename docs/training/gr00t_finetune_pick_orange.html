读图要点：

  • GR00T 是 flow-matching VLM-conditioned action expert 范式（同 π0.5 / SmolVLA / X-VLA / FastWAM），但 action expert 是 独立的 cross-attn DiT（不与 VLM 共享 KV，是从 VLM features 用 cross-attn 拉信息）
  • Multi-embodiment 关键：CategorySpecificLinear/MLP 让不同机器人（cat_id）有独立的 fc 权重（同 X-VLA 的 DomainAwareLinear 思路），embodiment_tag="new_embodiment" 会自动分配一个 cat_id
  • VLM 一次推理，DiT cross-attn 拉它的 features 做 N 步 denoise，跟 π0.5 一样高效
  • chunk_size=50, n_action_steps=50：1.67s open-loop @ 30Hz — 但 GR00T 50 步比 X-VLA 32 步还长，靠 multi-embodiment 预训练弥补
  • 训练 default：tune_llm=False, tune_visual=False, tune_projector=True, tune_diffusion_model=True — Eagle2 全冻，只训 projector + DiT action head

关键发现：N1.6 静态显存（Eagle 2.4B + DiT 600M trainable + bf16 master + Adam state）≈ 22 GB。激活根本不是瓶颈（per_step=8 和 2 都 OOM 在同一个 22.35 GB）。唯一能救的是 8-bit optim state（paged_adamw_8bit, bitsandbytes 0.49.2），把 32-bit Adam (~4.8 GB on 600M trainable) 压成 8-bit (~1.2 GB)，省 ~3 GB 给激活。

3.3.1 最终敲定配方（2026-05-20 v3）

# LeIsaac/scripts/finetune/gr00t/launch_finetune_ckpt.py 中固定写入：
config.model.backbone_trainable_params_fp32 = False  # save ~3 GB master fp32
config.training.gradient_checkpointing       = True  # save ~half activations
config.training.optim                        = "paged_adamw_8bit"  # save ~3 GB

# train.sh env：
GLOBAL_BATCH=16
GRAD_ACCUM=8                # per-step = 2
PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True

显存预算（实测峰值 24000 MiB / 24 GB）：

  • Eagle bf16 base (frozen, ~2.4B): ~4.8 GB
  • top4 LLM layers + DiT trainable bf16 (~600M): ~1.2 GB params + ~1.2 GB grads
  • paged_adamw_8bit state (~600M × 1 byte × 2): ~1.2 GB
  • Activations (grad-ckpt halved, per_step=2, chunk=50, 448×448): ~14-15 GB
  • 合计 ≈ 22.4 GB，4090 上跑得动但留白小（建议训练时不要开其他 GPU 进程）
📌 注意：训练时 4090 几乎打满（24000 MiB / 24576 MiB），不能并行跑 GR00T 推理 server 做 auto-eval。下游 closed-loop eval 必须等训练完成后串行跑（与 X-VLA workflow 一致）。
⚠️ exit 139 偶发 SIGSEGV：尝试 5 在 ckpt-500 save 完后进程 exit 139（无 Python traceback）。怀疑是 bnb 0.49.2 在 24GB 峰值时 paged 分配的 cuda kernel 偶发崩。HF Trainer auto-resume 完美兼容：删 ckpt 之外的孤儿文件后重启 train.sh，会自动从 latest checkpoint-N 续。
对应策略：把 train.sh 包一层 watchdog，detect exit 139 后自动重启（沿用 OpenVLA session 同款方案）。

3.5 Auto-eval-on-save 循环

训练 background 跑，同时 scripts/auto_sweep_gr00t_ckpts.sh 监控 $OUTPUT_DIR/checkpoint-*，每发现新 ckpt 就：

  1. 启动 server/start_server.sh --gr00t-only，GR00T_MODEL_PATH=<ckpt>
  2. 跑 6-round eval（EVAL_ROUNDS=6 EPISODE_LENGTH=60 MAX_ROUND_WALL_S=90）
  3. 解析 metrics JSON，append 一行到 logs/gr00t_ckpts.csv
  4. kill server，继续等下一个 ckpt

训练不中断；eval 串行在 server pause 间隙跑（每 ckpt ≈ 90s × 6 = 9 min 评测；训练每 500 step ≈ 25-30 min @ 4090，eval 不会撞车）。

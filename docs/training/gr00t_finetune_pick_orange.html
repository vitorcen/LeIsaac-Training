GR00T N1.5 / N1.6 微调计划

SO-101 PickOrange · LeIsaac benchmark champion (N1.6 🥇 2/3) · 2026-05-18

NVIDIA Isaac GR00T N1.5/N1.6 — Eagle2 VLM + multi-embodiment flow-matching DiT action head, ~3B params. Already top of the leaderboard via the LightwheelAI public ckpt; the next obvious lever is in-house LoRA / QLoRA finetune to push from 6/9 to 8/9 oranges.

0. TL;DR

GR00T-N1.6 是当前 leaderboard 🥇（LightwheelAI ckpt 实测 2/3 strict round, 6/9 oranges, 96s avg, 17.3 GB VRAM）。 架构：Eagle2-VLM (~2.4B, frozen) + Flow-matching Action DiT (~600M, trainable)，3B 总参数。 N1.5 是上一代（0/3, 4/9，但已 commit 落盘 8 颗其中 4 strict）；N1.6 用更新的 Eagle2.5 backbone + 优化 action head。

下一步：自己跑 LoRA / QLoRA 微调（lerobot config 原生支持 lora_rank），从 nvidia/GR00T-N1.6-3B base 出发，预期能从 2/3 → 3/3。

1. 架构核心

组件来源实现参数估算
Eagle2 VLM backboneNVIDIA Eagle 2 (2024)SigLIP-style vision encoder + Llama 3-class decoder，N1.5 用 Eagle2，N1.6 升级到 Eagle 2.5（eagle2_hg_model/modeling_eagle2_5_vl.py）；默认 tune_llm=False, tune_visual=False 全 frozen~2.4B
Visual / state projectorGR00T 自研把 VLM features + state 投到 action DiT 输入维；默认 tune_projector=True 可训~10M
MultiEmbodimentActionEncoderGR00T 自研（flow_matching_action_head.py:67）CategorySpecificLinear/MLP：每个 embodiment 有独立的 fc 权重（同 X-VLA DomainAwareLinear 思路），把 6/7-DOF action + 时间步投到 hidden~30M
Cross-attention DiT action headGR00T 自研（cross_attention_dit.py:204）N 层 BasicTransformerBlock：每层先 self-attn(action tokens)，再 cross-attn(action, VLM features)，AdaLayerNorm 注入 timestep；输出 noise/velocity prediction；默认 tune_diffusion_model=True 可训~500M
Flow-matching schedulerGR00T 自研continuous flow matching（同 π0.5/SmolVLA/X-VLA/FastWAM 家族）—
合计——~3B（VLM frozen, trainable ~540M）
cross-attn(action, VLM features) action=Q, VLM=K/V — 经典 DiT ⑤ AdaLayerNorm ⑥ MLP (4× expansion) 类似 DALL-E 2 / SD 3 的 cross-attn multi-embodiment 关键： CategorySpecific 让不同机器人有独立 norm Action decoder CategorySpecificMLP → (B, 50, 32) 取前 6 维 = SO-101 Flow-matching loss L = ||v_pred − (ε−a)||² 仅在 action tokens 推理 (inference) • x_T = N(0, 1) chunk=50 • VLM forward 1 次 　缓存 VLM features (重要!) • for i in range(steps): 　 DiT cross-attn → v_pred 　 x_t ← scheduler.step() → ~80-100ms / chunk @ 4090 • n_action_steps=50 整个 chunk 跑完再 query step_hz=30 / 50 = 0.6Hz query state cat_id VLM features → KV action_noisy + t N1.5 vs N1.6 差异 N1.5 (nvidia/GR00T-N1.5-3B): • Eagle2 vision • 我们 benchmark 0/3, 4/9 N1.6 (nvidia/GR00T-N1.6-3B): • Eagle 2.5 vision • image augmentations • 优化 action head • 🥇 2/3, 6/9 (currently)

读图要点：

  • GR00T 是 flow-matching VLM-conditioned action expert 范式（同 π0.5 / SmolVLA / X-VLA / FastWAM），但 action expert 是 独立的 cross-attn DiT（不与 VLM 共享 KV，是从 VLM features 用 cross-attn 拉信息）
  • Multi-embodiment 关键：CategorySpecificLinear/MLP 让不同机器人（cat_id）有独立的 fc 权重（同 X-VLA 的 DomainAwareLinear 思路），embodiment_tag="new_embodiment" 会自动分配一个 cat_id
  • VLM 一次推理，DiT cross-attn 拉它的 features 做 N 步 denoise，跟 π0.5 一样高效
  • chunk_size=50, n_action_steps=50：1.67s open-loop @ 30Hz — 但 GR00T 50 步比 X-VLA 32 步还长，靠 multi-embodiment 预训练弥补
  • 训练 default：tune_llm=False, tune_visual=False, tune_projector=True, tune_diffusion_model=True — Eagle2 全冻，只训 projector + DiT action head

3. 微调计划 — LoRA / QLoRA from N1.6 base

为什么 LoRA + N1.6 base 是合理下一步：
  • N1.6 公开 ckpt 已 2/3 → 离 3/3 只差关键样本细节
  • lerobot 原生支持 LoRA（configuration_groot.py 有 lora_rank/lora_alpha/lora_dropout 字段）
  • Eagle2 backbone frozen 不动 → 只训 projector + DiT 几百 M params
  • 预期：从 multi-embodiment generalist → SO-101 specialist，6/9 → 8/9 是合理目标

3.1 推荐配方：bf16 LoRA

config 字段推荐值理由
base_model_pathnvidia/GR00T-N1.6-3B当前 SOTA base，比 N1.5 强一档
chunk_size50保留默认（dataset 30Hz × 1.67s 训练上下文）
n_action_steps16（不要 50）消费前 16 步重 query：267ms 重规划周期，避免 1.67s open-loop 错过抓握时机
tune_llmFalse2.4B Eagle decoder 全冻
tune_visualFalsevision encoder 全冻
tune_projectorTrue~10M，必须训以适配 SO-101
tune_diffusion_modelTrue~500M action head，主要 trainable
lora_rank320 = 全参微调（更准但 VRAM 紧），32 = LoRA on DiT cross-attn q/v（节省 VRAM）
lora_alpha16默认 r=32 alpha=16，scale=0.5
lora_dropout0.1默认
lora_full_modelFalse仅挂 DiT 不挂 VLM
embodiment_tag"new_embodiment"自动分配新 cat_id；不要复用预训练 embodiment（会偏向其他机器人）
max_state_dim64默认；我们 6D state pad → 64
max_action_dim32默认；我们 6D action pad → 32
use_bf16True
optimizer_lr1e-4 (LoRA) / 2e-5 (full)LoRA 可大学习率
warmup_ratio0.05默认

3.2 GPU 资源估算（4090 24GB）

项bf16 LoRA (推荐)QLoRA NF4 (备选)全参 bf16 (拒)
Eagle2 base (frozen)~4.8 GB~1.3 GB~4.8 GB
DiT action head (trainable)~1.0 GB~1.0 GB~1.0 GB
LoRA params (r=32)~0.05 GB~0.05 GB—（全参）
Optimizer (LoRA only)~0.2 GB~0.2 GB~6 GB
Activations (batch=4, chunk=50)~8-10 GB~8-10 GB~12 GB
合计~14-16 GB ✅~10-12 GB ✅~24 GB ❌ OOM

推荐：bf16 LoRA。QLoRA 不必要（base 4.8 GB，bf16 头顶 16 GB 还安全）；QLoRA 反而会引入 bnb flaky 问题（vanilla OpenVLA 那 4 类 crash）。

3.3 训练 launcher 草稿

# LeIsaac/scripts/finetune/gr00t/train.sh

CONDA_ENV=lerobot \
DATASET_DIR=$HOME/work/isaaclab-experience/LeIsaac/datasets/raw/leisaac-pick-orange \
OUTPUT_DIR=$HOME/work/isaaclab-experience/LeIsaac/outputs/gr00t-n16-leisaac-pick-orange \
BASE_CKPT=nvidia/GR00T-N1.6-3B \
\
conda run -n "$CONDA_ENV" --no-capture-output \
    python -m lerobot.scripts.lerobot_train \
        --policy.type=groot \
        --policy.path="$BASE_CKPT" \
        --policy.base_model_path="$BASE_CKPT" \
        --policy.chunk_size=50 \
        --policy.n_action_steps=16 \
        --policy.tune_llm=false --policy.tune_visual=false \
        --policy.tune_projector=true --policy.tune_diffusion_model=true \
        --policy.lora_rank=32 --policy.lora_alpha=16 --policy.lora_dropout=0.1 \
        --policy.lora_full_model=false \
        --policy.embodiment_tag=new_embodiment \
        --policy.use_bf16=true \
        --policy.optimizer_lr=1e-4 \
        --policy.warmup_ratio=0.05 \
        --dataset.repo_id=local \
        --dataset.root="$DATASET_DIR" \
        --output_dir="$OUTPUT_DIR" \
        --batch_size=4 --num_workers=4 \
        --steps=10000 --save_freq=500

4. 风险清单

风险概率影响缓解
🟡 N1.6 base 的 cat_id 预训练偏置盖过我们 LoRA中中用 new_embodiment tag 新 cat_id；避免选 SO-101 已知 tag（如果有）
🟡 chunk=50 open-loop 1.67s 抓握时机不稳已观察中n_action_steps=16 重规划；如还不够再砍到 8
🟡 SO-101 6D state pad 到 64 D 引入大量 0 通道—低同其他 baseline，pad 默认无害
🟢 GPU OOM低—bf16 LoRA 头顶 16 GB
🟢 lerobot trainer 兼容低低同 dataset 已跑过 SmolVLA / DP / ACT
🟢 Eagle2 vision encoder 跟我们 480×640 兼容低低config 默认 image_size=(224, 224)，processor 会 resize

6. 与 X-VLA 的取舍

维度X-VLAGR00T-N1.6 LoRA
Starting point0/3, 0/9 (cold)2/3, 6/9 (warm)
架构新奇度soft-prompted Transformer (科研味浓)flow-matching DiT (成熟)
工程成本需修 action space + domain reinit0 修改（lerobot 原生）
预期 leaderboard1-2/3（如果一切顺利）2-3/3（最差也是当前水平）
失败 fallbackX-VLA 半完成仍然有 N1.6 base 兜底

结论：GR00T-N1.6 LoRA 比 X-VLA 更值得先做 — 起点高、工程便宜、上限明确。X-VLA 作为"架构探索 + 多 family benchmark 多样性"价值，但 production 优先级低于 GR00T finetune。
